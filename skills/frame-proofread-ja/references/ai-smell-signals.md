# AI-smell シグナルテーブル

Pass C（AIくさい表現検出）の照合基準。`proofread-ng-check.py` が検出できないコンテキスト依存パターンを AI で補完するときに参照する。

## A. 列挙の前置き（ChatGPT 固有度: 高）

| 典型例 | 判定理由 |
|---|---|
| 「以下の点が挙げられます」 | 内容より形式を優先する |
| 「いくつかのポイントがあります」 | 読者に何も伝えない前置き |
| 「次の〜に注意が必要です」 | 内容を先送りする |
| 「〜について見ていきましょう」 | ガイド口調の誘導 |
| 「〜を深掘りしていきます」 | 同上 |

## B. 冗長な強調（AI全般）

| 典型例 | 判定理由 |
|---|---|
| 「重要なのは〜です」 | 強調が自己目的化している |
| 「〜することが大切です」 | 結論を弱める表現 |
| 「非常に重要な〜」「とても大切な〜」 | 副詞による水増し |
| 「〜が求められます」 | 主体がない受け身の強調 |
| 「〜に注意が必要です」 | 同上 |

## C. 受け身の逃げ（AI全般）

| 典型例 | 判定理由 |
|---|---|
| 「〜と考えられます」 | 根拠なく断定を回避 |
| 「〜という側面もあります」 | 立場を取らずにぼかす |
| 「〜とも言えるでしょう」 | 同上 |
| 「〜場合があります」「〜ことがあります」の多用 | 過度な例外留保 |

## D. 空中の締め（ChatGPT 固有度: 高）

| 典型例 | 判定理由 |
|---|---|
| 「〜を心がけましょう」「〜していきましょう」 | 読者への命令で終わる空虚な締め |
| 「いかがでしたか？」 | 日本語 ChatGPT 記事の定番 |
| 「参考になれば幸いです」「ぜひ活用してください」 | 内容のない礼儀的締め |
| 「本記事では〜について解説しました」 | 内容を繰り返すだけのまとめ |

## E. 言い換え繰り返し（AI全般）

| 典型例 | 判定理由 |
|---|---|
| 「つまり」「言い換えると」「すなわち」で直前の文を言い換える | 同義反復で水増し |
| 「〜です。〜です。〜です。」の体言止め連発 | 単調なリズムの反復 |

## F. 構造的マーカーの過剰使用（ChatGPT 固有度: 中）

| 典型例 | 判定理由 |
|---|---|
| 段落先頭に「まず」「次に」「そして」「最後に」を必ず置く | 接続詞で構造を強制する |
| 本文が短い文書での見出しの多用 | 内容のない見出しの乱立 |
| 説明できる内容を全て箇条書きにする | 文章で書くべきものを箇条書きへ逃げる |

## G. 肯定から始まる反論回避（ChatGPT 固有度: 高）

| 典型例 | 判定理由 |
|---|---|
| 「もちろん〜ですが」「確かに〜ですが」 | 反論を形式的に認めてから無効化する |
| 「おっしゃる通りですが」 | 同上 |

## H. 曖昧修飾語（AI全般）

| 典型例 | 判定理由 |
|---|---|
| 「さまざまな〜」「多くの〜」「多岐にわたる〜」 | 具体性を避けた曖昧な量の表現 |
| 「包括的な〜」「総合的な〜」 | 中身を示さない形容詞 |
| 「近年〜が注目されています」「ますます重要に〜」 | 出典も根拠もない抽象的な一般論の冒頭 |

## I. 接続詞の全文頭固定（AI全般）

| 典型例 | 判定理由 |
|---|---|
| 段落・文の先頭に「さらに」「また」「一方で」「しかしながら」を機械的に配置 | 論理的な流れではなく形式的な接続詞で構造を作る |
| 「そのため」「したがって」の多用 | 因果が成立していない箇所への誤使用 |

## J. 評価定型句（AI全般）

| 典型例 | 判定理由 |
|---|---|
| 「〜が鍵となります」「〜が必要不可欠です」 | 強い断言のように見えて内容が薄い |
| 「〜することが重要です」（B の「重要なのは〜」の変形） | 主体を消した義務表現 |
| 「ぜひ〜してみてください」「〜することをおすすめします」 | 勧誘定型。Dの空中の締めと同類 |

## K. 形式書き言葉（AI全般）

| 典型例 | 判定理由 |
|---|---|
| 「〜においては」「〜を踏まえ」「〜の観点から」 | 官公庁文書調の形式表現。Webの文脈では浮く |
| 「〜することが可能です」 | 「〜できます」で十分な箇所への回りくどい言い換え |
| 「〜と言われています」「〜が望ましいでしょう」 | 出典を出さずに権威に委ねる曖昧な断定 |
