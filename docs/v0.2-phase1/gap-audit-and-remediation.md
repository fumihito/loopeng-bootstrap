# 乖離監査と補完指示

対象: Phase 1 再発行指示および自己適用化手順の監査・補完。

## 役割

- 既存実装、テスト、文書、ゲートの間にある乖離を特定する。
- 乖離を埋めるために必要な追加作業を、順序と依存関係つきで明示する。
- 実装済みの前提を勝手に置き換えず、現物の状態を基準にする。

## 監査観点

- 仕様に書かれた条件が実装とテストに同一の形で反映されているか。
- 空検証や vacuous test が混入していないか。
- 完了宣言が、実際の gate PASS で裏づけられているか。
- 参照文書が散在していないか。リポジトリ内の文書が唯一の参照になっているか。

## 補完方針

1. まず実在の artifact 名と gate 実装を突き合わせる。
2. 次にテストを、存在しない断片ではなく実在資材へ向け直す。
3. 最後に文書と完了報告を gate 結果に結びつける。

## 禁止

- 期待結果をテストの名前だけで正当化すること。
- 未検証の宣言を完了扱いすること。
- ゲートの変更禁止条件を無視して実装側を改変すること。
